# 产品需求文档 (PRD) - v0.4.0 性能与体验完善

**产品名称**: 待办事项列表应用 (To-Do List App)
**文档版本**: v0.4.0
**创建日期**: 2026-04-01
**产品经理**: 小虾米
**版本性质**: 性能优化 + 视觉收尾 + 体验增强，不新增核心功能

---

## 1. 背景与目标

### 1.1 问题描述

当前 v0.3.0 已完成界面美化升级（Indigo 渐变主题 + Soft UI 风格），但存在以下遗留问题：

**性能层面**：
- v0.3.0 原规划的性能优化（虚拟滚动、搜索优化、滚动位置记忆）被推迟，至今未实现
- 当任务量超过 500 条时，页面渲染开始出现明显卡顿
- 首屏加载大量任务时，DOM 节点过多导致渲染阻塞
- 搜索和过滤操作在大数据量下响应超过 100ms

**视觉收尾**：
- 过滤器 Tab 切换缺少滑块动画，与整体动效水平不一致
- 优先级选择器切换缺少滑块动画，体验略显生硬
- 优先级指示仅有左侧竖条，缺少小圆点等辅助视觉元素（US-308）
- 空状态 SVG 插画仍使用灰色调，与 Indigo 主题不匹配

### 1.2 目标用户

与 v0.3.0 一致 -- 个人用户，主要在浏览器中管理日常待办事项。

### 1.3 产品目标（可量化）

| 指标 | 当前值（v0.3.0） | 目标值（v0.4.0） |
|------|------------------|------------------|
| 首屏加载时间（10000 条任务） | 约 9s（全量渲染） | < 2s（虚拟滚动） |
| 滚动帧率（10000 条任务） | 卡顿严重 | >= 55fps |
| 搜索/过滤响应时间（10000 条） | > 500ms | < 100ms |
| 任务操作响应时间 | 100-200ms（大数据量） | < 100ms |
| v0.3.0 遗留视觉项完成率 | 0% | 100% |

## 3. 功能需求

### 3.1 虚拟滚动列表

- **描述**：只渲染可视区域内的任务项，滚动时动态加载/卸载 DOM 节点，解决大数据量下的性能问题
- **主流程**：
  1. 应用加载时，仅渲染可视区域内的任务项（约 20-30 条）
  2. 用户滚动时，计算滚动位置，动态更新渲染窗口
  3. 离开可视区域的任务项被卸载，新进入的任务项被挂载
  4. 滚动条高度反映完整列表长度，用户可拖动到任意位置
- **异常流程**：
  - 任务数量较少（< 50 条）时，降级为普通列表渲染，避免虚拟滚动的额外开销
  - 快速滚动时可能出现短暂白屏，通过增加渲染缓冲区（上下各额外渲染 5 条）来缓解
- **验收标准**：
  - 支持 10000+ 任务流畅滚动，帧率 >= 55fps
  - 首屏加载时间 < 2s（10000 条任务场景）
  - 滚动到任意位置后，任务内容显示正确
  - 任务增删改后，列表正确更新且滚动位置保持
  - 与过滤、搜索功能联动正确

### 3.2 搜索/过滤性能优化

- **描述**：优化搜索和过滤的数据处理逻辑，确保大数据量下的操作响应速度
- **主流程**：
  1. 搜索输入使用防抖（debounce 150ms），避免每次按键都触发过滤计算
  2. 过滤结果使用 `computed` 缓存，依赖不变时不重复计算
  3. 搜索算法优化为大小写不敏感的前缀匹配
  4. 过滤/搜索结果与虚拟滚动联动，只渲染可见部分
- **异常流程**：
  - 搜索无结果时，正确显示空状态
  - 过滤切换时，滚动位置重置到顶部
- **验收标准**：
  - 10000 条任务场景下，搜索响应 < 100ms
  - 过滤切换响应 < 100ms
  - 搜索与过滤可叠加使用
  - 虚拟滚动在过滤/搜索结果上正常工作

### 3.3 滚动位置记忆

- **描述**：记住用户的滚动位置，在刷新页面或切换过滤器后恢复
- **主流程**：
  1. 用户滚动列表时，记录当前滚动偏移量到 session storage
  2. 页面刷新或重新加载时，从 session storage 读取偏移量并恢复
  3. 切换过滤器后，保持当前滚动位置（如果位置在有效范围内）
  4. 执行搜索后，自动滚动到第一个匹配结果
- **异常流程**：
  - 记录的滚动位置超出当前列表范围时（如任务被删除），重置到顶部
  - session storage 不可用时，静默降级，不影响功能
- **验收标准**：
  - 刷新页面后，滚动位置恢复到之前的位置（误差 < 2 个任务项高度）
  - 切换过滤器后，滚动位置保持不变（有效范围内）
  - 搜索后，自动定位到第一个匹配结果
  - 位置恢复过程无明显闪烁

### 3.4 过滤器 Tab 滑块动画

- **描述**：为过滤器 Tab 切换增加背景色滑块过渡动画，从 v0.3.0 的渐变过渡升级为滑动效果
- **主流程**：
  1. 过滤器容器内增加一个绝对定位的滑块元素
  2. 切换 Tab 时，滑块通过 `transform: translateX()` 从当前位置滑动到目标位置
  3. 滑动动画时长 200ms，使用 `ease-in-out` 缓动
  4. 滑块背景色为白色（Header 内）或主色浅色系
- **异常流程**：
  - `prefers-reduced-motion` 下禁用滑块位移动画，仅保留背景色过渡
- **验收标准**：
  - 切换 Tab 时滑块平滑滑动到目标位置
  - 滑动方向正确（左→右 / 右→左）
  - 动画流畅无闪烁，帧率 >= 55fps
  - 不影响 v-model 绑定和 aria 属性
  - `prefers-reduced-motion` 下降级为背景色过渡

### 3.5 优先级选择器滑块动画

- **描述**：为优先级选择器切换增加背景色滑块过渡动画
- **主流程**：
  1. 优先级选择器容器内增加一个绝对定位的滑块元素
  2. 切换优先级时，滑块通过 `transform: translateX()` 滑动到目标位置
  3. 滑块背景色跟随目标优先级的背景色
  4. 滑动动画时长 200ms，使用 `ease-in-out` 缓动
- **异常流程**：
  - `prefers-reduced-motion` 下禁用滑块位移动画，仅保留背景色过渡
- **验收标准**：
  - 切换优先级时滑块平滑滑动到目标位置
  - 滑块背景色与目标优先级色系一致
  - 不影响 radiogroup/aria 语义
  - `prefers-reduced-motion` 下降级为背景色过渡

### 3.6 优先级指示小圆点

- **描述**：在任务卡片上增加优先级小圆点指示器，提供额外的视觉标识
- **主流程**：
  1. 在任务文字左侧（复选框右侧）增加一个小圆点
  2. 圆点颜色与任务优先级对应（高=红、中=橙、低=绿）
  3. 圆点尺寸 6px，与文字垂直居中
  4. 已完成任务的小圆点变为灰色
- **异常流程**：无
- **验收标准**：
  - 圆点颜色与优先级正确对应
  - 圆点在任务文字左侧，不干扰文字阅读
  - 已完成任务圆点变灰
  - 移动端圆点清晰可见

### 3.7 空状态插图配色更新

- **描述**：将空状态 SVG 插画的灰色调更新为 Indigo 主题色系
- **主流程**：
  1. SVG 插图的线条色从灰色改为 Indigo 主色 `#4F46E5`
  2. 辅助线条从浅灰改为 `#C7D2FE`（Indigo 200）
  3. 填充区域从灰色改为 `#EEF2FF`（Indigo 50）
  4. 保持 SVG 路径不变，仅更新 fill/stroke 属性
- **异常流程**：无
- **验收标准**：
  - SVG 配色与 Indigo 主题协调
  - SVG 在浅色背景上清晰可见
  - 浮动动画正常工作
  - 移动端缩小后仍然清晰

## 7. 假设与约束

### 7.1 假设

- [假设 1] 虚拟滚动使用成熟的开源库（如 vue-virtual-scroller），不自研，以控制复杂度
- [假设 2] 任务项高度基本固定（单行文本），虚拟滚动可采用固定高度模式，简化实现
- [假设 3] v0.3.0 的 CSS 变量体系完整，滑块动画等视觉需求不需要新增 CSS 变量
- [假设 4] 搜索/过滤优化在虚拟滚动基础上进行，先完成虚拟滚动再优化搜索
- [假设 5] 滚动位置记忆使用 session storage（标签页级别），不使用 local storage（不需要跨标签页持久化）

### 7.2 约束

- [技术约束] 虚拟滚动库必须兼容 Vue 3 + TypeScript + Pinia
- [技术约束] 不改变现有数据模型（Task 接口），虚拟滚动是纯渲染层优化
- [兼容约束] 优先级小圆点等新增 DOM 元素不能影响虚拟滚动的性能
- [性能约束] 所有新增动画（滑块）仅使用 transform/opacity，不触发 layout/paint
- [时间约束] 预计开发周期 5-7 天

4. 刷新页面后滚动位置恢复（误差 < 2 个任务项高度）
5. 过滤器 Tab 切换有滑块滑动动画
6. 优先级选择器切换有滑块滑动动画
7. 所有新增动画遵循 prefers-reduced-motion
8. 现有功能（增删改查、搜索过滤、优先级）回归测试通过

### 8.2 可选验收项

9. 任务卡片上有优先级小圆点指示器
10. 空状态 SVG 插画配色更新为 Indigo 色系

### 8.3 不验收项

- 深色模式
- 任务分类/标签
- 任务截止日期
- 数据导出/导入

## 9. 风险与依赖

### 9.1 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 虚拟滚动库与现有组件不兼容 | 开发进度 | 选型阶段进行 PoC 验证，提前发现集成问题 |
| 虚拟滚动导致滚动条高度计算不准确 | 用户体验 | 使用固定高度模式，避免动态高度计算的复杂性 |
| 滑块动画在虚拟滚动环境下性能问题 | 帧率下降 | 滑块仅在用户操作时触发，不涉及列表渲染 |
| 滚动位置记忆在虚拟滚动下精度不足 | 位置偏差 | 记录任务索引而非像素偏移，通过索引计算位置 |
| 搜索过滤后虚拟滚动状态不一致 | 列表显示异常 | 搜索过滤后重置虚拟滚动状态，强制重新计算 |

### 9.2 依赖

- 依赖软件架构师完成虚拟滚动技术选型（ARCH-401）
- 依赖 UX/UI 设计师完成滑块动画和视觉增强的设计定义（DES-401 ~ DES-404）
- 依赖前端工程师确认虚拟滚动不影响现有功能
