## 1. 系统安全边界

### 1.1 外部入口
- **用户输入接口**:
  - 任务文本输入框 (TaskInput.vue)
  - 搜索框 (SearchBar.vue)
  - 优先级选择器 (PrioritySelector.vue)
  - 任务编辑模式
- **数据持久化接口**:
  - LocalStorage API
  - Memory Storage (降级方案)

### 1.2 敏感数据资产
- **用户数据**:
  - 任务描述 (可能包含个人信息、密码等敏感内容)
  - 任务创建时间/更新时间 (用户行为模式)
  - 任务优先级 (用户工作习惯)
- **元数据**:
  - LocalStorage Schema 版本
  - 过滤器状态

### 1.3 信任边界划分
```
┌─────────────────────────────────────┐
│   不可信区域: 用户浏览器环境         │
│   - 潜在恶意扩展                     │
│   - XSS 攻击向量                     │
│   - 本地存储访问风险                 │
└─────────────────────────────────────┘
              ↑
              │ 数据输入边界
┌─────────────────────────────────────┐
│   应用边界: Vue 3 SPA               │
│   - UI 组件层                        │
│   - 状态管理层 (Pinia)               │
│   - 业务逻辑层 (Services)            │
└─────────────────────────────────────┘
              ↑
              │ 数据持久化边界
┌─────────────────────────────────────┐
│   存储边界: LocalStorage            │
│   - 同域名下所有脚本可访问            │
│   - 容量限制: 5-10MB                 │
│   - 明文存储                         │
└─────────────────────────────────────┘
```

## 2. STRIDE 威胁分析

| 威胁类型 | 攻击场景 | 影响范围 | 优先级 |
|---------|---------|---------|--------|
| **Spoofing (身份伪造)** | 1. 恶意脚本通过注入代码伪造用户操作<br>2. 跨站脚本攻击伪造用户点击行为 | - 未授权任务创建/删除<br>- 用户行为数据篡改 | **P1** |
| **Tampering (数据篡改)** | 1. XSS 攻击修改 LocalStorage 数据<br>2. 浏览器扩展直接修改 LocalStorage<br>3. 中间人攻击替换 JS 文件(无 HTTPS) | - 任务数据丢失/篡改<br>- 应用状态异常<br>- 数据完整性破坏 | **P0** |
| **Repudiation (抵赖)** | 1. 用户否认创建某任务(无审计日志)<br>2. 操作无时间戳记录 | - 无法追溯操作历史<br>- 协作场景下责任界定困难 | **P2** |
| **Information Disclosure (信息泄露)** | 1. LocalStorage 明文存储敏感信息<br>2. XSS 攻击窃取 LocalStorage 数据<br>3. 浏览器缓存泄露任务内容<br>4. 搜索关键词通过侧信道攻击泄露 | - 隐私信息泄露<br>- 用户行为模式暴露<br>- 敏感任务内容泄露 | **P0** |
| **Denial of Service (拒绝服务)** | 1. 恶意创建超长任务文本导致存储溢出<br>2. 快速创建大量任务(10000+)导致性能崩溃<br>3. 恶意脚本清空 LocalStorage | - 应用不可用<br>- 数据丢失<br>- 用户体验下降 | **P1** |
| **Elevation of Privilege (权限提升)** | 1. XSS 攻击获取 LocalStorage 完整访问权限<br>2. 原型链污染绕过输入验证 | - 完全控制应用数据<br>- 执行任意 JavaScript | **P0** |

## 3. v0.2.0 功能特定威胁分析

### 3.1 优先级选择功能 (新增)
| 威胁类型 | 攻击场景 | 风险等级 |
|---------|---------|---------|
| Tampering | 恶意脚本将所有任务优先级改为 'high',干扰用户工作 | 中 |
| XSS | 优先级值未验证,注入非预期值导致逻辑错误 | 低 |

### 3.2 搜索功能 (新增)
| 威胁类型 | 攻击场景 | 风险等级 |
|---------|---------|---------|
| XSS | 搜索关键词使用 `v-html` 渲染高亮,注入恶意脚本 | **高** |
| Information Disclosure | 搜索预测泄露用户输入内容 | 低 |
| DoS | 输入超长搜索字符串(如10000字符)导致性能下降 | 中 |

## 4. 安全加固要求

### P0 — 必须实现 (高风险,阻塞上线)

#### 4.1 XSS 防护
- [ ] **SR-001**: 搜索高亮功能必须对用户输入进行 HTML 转义
  - 实现: `highlightMatch()` 函数使用 DOMPurify 或手动转义 `<`, `>`, `&`, `"`, `'`
  - 禁止使用 `v-html` 渲染未经转义的用户输入
  - 单元测试: 验证 `<script>alert('XSS')</script>` 被正确转义

- [ ] **SR-002**: 所有用户输入必须经过 Vue 3 的自动转义
  - 确保 TaskInput、SearchBar 组件不使用 `v-html`
  - 确保 TaskItem 组件渲染任务文本时不使用 `v-html`

#### 4.2 输入验证强化
- [ ] **SR-003**: 搜索关键词长度限制
  - 限制搜索关键词最大长度为 100 字符
  - 超长输入截断或拒绝处理

- [ ] **SR-004**: 优先级值验证
  - `PrioritySelector` 组件必须验证 `modelValue` 为合法值 (`'high' | 'medium' | 'low'`)
  - Store 层对 `priority` 字段进行运行时类型检查

#### 4.3 LocalStorage 数据完整性
- [ ] **SR-005**: LocalSchema 版本控制
  - 实现 `version` 字段,用于数据迁移
  - 加载时验证版本号,不兼容时显示警告或降级处理

- [ ] **SR-006**: LocalStorage 数据签名验证
  - 使用 HMAC-SHA256 对存储数据进行签名
  - 加载时验证签名,防止篡改
  - 密钥存储在代码中(混淆)或用户本地(首次生成)

#### 4.4 HTTPS 强制
- [ ] **SR-007**: 生产环境强制 HTTPS
  - 部署配置启用 HSTS (HTTP Strict Transport Security)
  - 防止中间人攻击替换 JS 文件

---

### P1 — 建议实现 (中风险,下个版本完成)

#### 5.1 性能与可用性
- [ ] **SR-101**: 任务数量限制
  - 限制最大任务数为 10000
  - 接近限制时提示用户删除旧任务

- [ ] **SR-102**: 搜索防抖时间优化
  - 当前 200ms,建议调整为 300ms 以减少计算频率
  - 或实现指数退避策略

#### 5.2 数据隐私增强
- [ ] **SR-103**: 敏感数据提示
  - 首次使用时提示: "请不要在任务中存储密码、银行卡号等敏感信息"

- [ ] **SR-104**: LocalStorage 加密 (可选)
  - 使用 Web Crypto API (AES-GCM) 加密存储数据
  - 密钥从用户输入派生 (如主密码) 或存储在 SessionStorage

#### 5.3 安全监控
- [ ] **SR-105**: 错误日志记录
  - 记录所有异常操作(如验证失败、存储失败)
  - 不记录敏感内容,仅记录操作类型和频率

- [ ] **SR-106**: CSP (Content Security Policy) 头
  - 部署时配置 CSP 策略,限制脚本来源
  - 示例: `default-src 'self'; script-src 'self'`

---

### P2 — 后续跟进 (低风险,长期优化)

#### 6.1 审计与追溯
- [ ] **SR-201**: 操作审计日志 (本地)
  - 记录任务创建/更新/删除操作
  - 记录时间戳和操作类型
  - 用于抵赖防护 (Repudiation)

#### 6.2 安全测试
- [ ] **SR-202**: 自动化安全测试
  - 集成 ESLint 插件检测潜在 XSS 风险
  - 使用 npm audit 检测依赖漏洞

#### 6.3 用户教育
- [ ] **SR-203**: 隐私政策文档
  - 说明数据仅存储在本地
  - 提示不要在公共设备使用
  - 说明浏览器清理会清除数据

## 6. 风险等级说明

| 风险等级 | 定义 | 上线建议 |
|---------|------|---------|
| **P0 (高)** | 可导致数据泄露、数据丢失、应用被完全控制 | **必须修复后才能上线** |
| **P1 (中)** | 可导致用户体验下降、部分功能异常 | **建议修复后上线,可后续版本完善** |
| **P2 (低)** | 可通过用户行为规避,影响范围有限 | **可后续版本优化** |
