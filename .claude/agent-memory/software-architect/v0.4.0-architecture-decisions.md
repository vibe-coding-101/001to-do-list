## v0.4.0 架构设计决策

### 虚拟滚动技术选型

**决策**: 选择 @tanstack/vue-virtual (v3.13.x) 作为虚拟滚动方案

**理由**:
- 稳定发布（非 beta），TanStack 团队持续维护（最近更新 2026-03-25）
- 原生 TypeScript 编写，零类型补丁成本
- Headless 设计（composable API），不侵入现有 DOM 结构
- ~2.5KB gzip，最轻量方案之一
- Composition API 原生支持，与 Pinia 响应式系统无缝对接

**排除方案**:
- vue-virtual-scroller: Vue 3 版本长期 beta（2.0.0-beta.x 已 2 年+），非原生 TS
- vue3-virtual-scroll-list: 停止维护（3 年未更新），无 TS 支持

**Why**: v0.2.0 遗留的"暂不实现虚拟滚动"待确认项，在 v0.4.0 正式解决

### 集成策略

**决策**: Headless composable 集成到 TaskList.vue，替换 TransitionGroup

**理由**:
- useVirtualizer composable 直接消费 filteredTasks computed
- getItemKey 使用 task.id 确保增删改时 DOM 正确复用
- 过滤/搜索切换时通过 scrollToIndex(0) 重置

**降级策略**: 任务数 < 50 时自动回退到 v-for + TransitionGroup 全量渲染

### 关键布局约束

**决策**: App.vue .main__content 需设置 overflow: hidden，虚拟滚动在 TaskList 内部容器触发

**Why**: 虚拟滚动需要固定高度的滚动容器，不能依赖页面级滚动

**How to apply**: FE-401 实现时需同时修改 App.vue 和 TaskList.vue 的 CSS 布局

### 待确认项

1. TaskItem 编辑模式高度固定方案（建议固定容器高度 + 内部溢出滚动）
2. App.vue overflow: hidden 是否影响 .input-section sticky bottom 定位
3. 虚拟滚动阈值 VIRTUAL_THRESHOLD 默认 50，可按实测调整
