# 产品需求文档 (PRD) - v0.3.0 界面美化升级

**产品名称**: 待办事项列表应用 (To-Do List App)
**文档版本**: v0.3.0
**创建日期**: 2026-04-01
**产品经理**: 小虾米
**版本性质**: 纯视觉/UI 升级，不改变功能逻辑

---

## 1. 背景与目标

### 1.1 问题描述

当前 v0.2.0 界面完成了全部核心功能（增删改查、优先级、搜索过滤），但视觉呈现停留在"功能性原型"阶段：

- **配色平淡**：主色 #3b82f6 纯色填充，缺少渐变与层次感，白底 + 灰边框显得单调
- **视觉层次不足**：header、任务卡片、输入区三个区域几乎只有边框区分，缺乏纵深和空间节奏
- **缺乏品牌感**：标题纯文字、无装饰，整体像框架默认样式，无产品辨识度
- **微交互粗糙**：hover/active 只有简单的 translateY 和 opacity 变化，缺乏弹性、形变等现代质感
- **空状态单薄**：SVG 插图线条简单，文字层级单一，无法在用户首次进入时产生好感
- **移动端体验割裂**：header 堆叠后高度过大，列表区被挤压，底部输入区与内容区无视觉过渡

### 1.2 目标用户

与 v0.2.0 一致 -- 个人用户，主要在浏览器中管理日常待办事项。

### 1.3 产品目标（可量化）

| 指标 | 当前值（v0.2.0） | 目标值（v0.3.0） |
|------|------------------|------------------|
| Lighthouse 视觉稳定性 (CLS) | 未测量 | < 0.1 |
| 首屏视觉完整时间 (FCP) | 未测量 | 不劣化（< 2s） |
| 用户主观满意度 | 功能可用但视觉平淡 | 现代感、精致感显著提升 |
| 设计一致性 | 各组件风格分散 | 统一设计语言，变量覆盖率 100% |

## 3. 功能需求

> 以下所有需求均为纯 CSS/样式/动画层面的改进，不改变任何功能逻辑、数据流或组件 props/events。

### 3.1 全局配色方案升级

- **描述**：替换当前纯色主色方案，引入渐变主色、更丰富的中性色层级、以及统一的圆角/阴影体系
- **主流程**：
  1. 在 `variables.css` 中更新 CSS 变量定义
  2. 主色从纯色 `#3b82f6` 升级为渐变（方向、色值由设计规范确定）
  3. 引入新增色彩变量：渐变起止色、表面色层级（surface-1/surface-2）、分隔线色
  4. 更新阴影层级，增加更柔和的 `box-shadow` 曲线
- **异常流程**：无
- **验收标准**：
  - 所有颜色通过 CSS 变量引用，无硬编码色值
  - 渐变在 Chrome / Firefox / Safari / Edge 最新版渲染一致
  - 所有文字与背景对比度 >= 4.5:1 (WCAG AA)
  - 深色文字在渐变背景上可读性良好

### 3.2 Header 视觉升级

- **描述**：将 header 从白底 + 1px 边框升级为渐变背景 + 毛玻璃效果的顶部栏，增强品牌感和空间纵深
- **主流程**：
  1. header 背景替换为渐变（从主色系到辅助色的线性渐变）
  2. 添加 `backdrop-filter: blur()` 毛玻璃效果
  3. 标题文字颜色适配新背景（白色或深色，保证对比度）
  4. header 下方阴影从 `shadow-1` 升级为 `shadow-2` 或更柔和的扩散阴影
- **异常流程**：
  - `backdrop-filter` 不支持的浏览器降级为半透明纯色背景
- **验收标准**：
  - header 渐变背景覆盖全部宽度
  - sticky 滚动时毛玻璃效果正常
  - 标题文字在渐变背景上清晰可读
  - `backdrop-filter` 降级方案在 Firefox 旧版等环境可用

### 3.3 任务卡片 (TaskItem) 视觉升级

- **描述**：优化任务卡片的视觉层次、hover 效果、完成态区分和编辑态样式
- **主流程**：
  1. 卡片默认态：增加左侧彩色竖条（根据优先级着色），圆角从 8px 增大到 12px
  2. 卡片阴影：默认态使用更柔和的阴影，hover 态阴影加深 + 轻微上浮 + 边框高亮
  3. 完成态：卡片背景变淡，左侧竖条变灰，整体降低饱和度（而非仅 opacity）
  4. 编辑态：边框渐变色而非纯色，内区域背景略有区分
- **异常流程**：无
- **验收标准**：
  - 卡片默认态有明确的视觉层级（左侧竖条 + 阴影）
  - hover 时卡片有明显但不过度的上浮效果（translateY 不超过 3px）
  - 完成态与未完成态视觉区分明显
  - 高/中/低优先级左侧竖条颜色不同
  - 卡片间距一致（gap 从 8px 调整为 10-12px）

### 3.4 复选框 (Checkbox) 样式升级

- **描述**：将复选框从方形边框升级为圆形，选中态增加弹性缩放动画和对勾绘制动画
- **主流程**：
  1. 复选框圆角从 4px 改为 `border-radius: 50%`
  2. 选中态背景色使用渐变而非纯色
  3. 选中瞬间增加 `scale(0.8) -> scale(1.1) -> scale(1.0)` 弹性动画
  4. 对勾 SVG 使用 `stroke-dashoffset` 动画绘制
- **异常流程**：无
- **验收标准**：
  - 复选框为正圆形（20x20px）
  - 选中动画在 300ms 内完成
  - `prefers-reduced-motion` 下禁用弹性动画，仅保留颜色变化
  - focus-visible 状态可见（外发光）

### 3.5 添加按钮 (Add Button) 视觉升级

- **描述**：将添加按钮从纯色方形升级为渐变 + 圆角 + 弹性点击反馈
- **主流程**：
  1. 按钮背景从 `#3b82f6` 纯色改为渐变
  2. 圆角从 4px 增大到 `border-radius: 12px`
  3. hover 态增加轻微发光（box-shadow 扩散）
  4. active 态增加 `scale(0.92)` 收缩效果
  5. 禁用态使用灰色渐变或降低透明度
- **异常流程**：无
- **验收标准**：
  - 按钮渐变与 header 渐变同源
  - hover/active 反馈流畅无闪烁
  - 按钮尺寸不变（40x40px 桌面 / 44x44px 移动端）
  - 图标在渐变背景上清晰可见

### 3.6 输入框 (Input Field) 样式升级

- **描述**：优化输入框的边框、聚焦态光效和 placeholder 样式
- **主流程**：
  1. 默认态边框从硬色 `#e5e7eb` 调整为更柔和的色调
  2. 聚焦态边框使用渐变色或主色调，外发光半径从 3px 增大到 4px
  3. placeholder 颜色调浅，使用斜体或更小的字号以区分
  4. 输入框圆角从 4px 增大到 8px
- **异常流程**：无
- **验收标准**：
  - 聚焦态视觉反馈明显但不刺眼
  - 输入框高度不变（40px 桌面 / 44px 移动端）
  - 聚焦动画时长 200ms

### 3.7 过滤器 Tab (FilterTabs) 视觉升级

- **描述**：将过滤器从底线指示器样式升级为胶囊/分段控制器样式
- **主流程**：
  1. 选中态从底部线条改为背景色填充（主色浅色系背景 + 主色文字）
  2. 增加 `border-radius: 8px` 圆角背景
  3. 切换时增加滑动过渡动画（背景色块从左到右/右到左平移）
  4. 过滤器整体容器添加浅灰背景底色
- **异常流程**：无
- **验收标准**：
  - 选中态与未选中态区分明确
  - 切换动画平滑（300ms）
  - 三个 Tab 等宽
  - 不影响 v-model 绑定和 aria 属性

### 3.8 搜索框 (SearchBox) 样式升级

- **描述**：优化搜索框的圆角、聚焦光效和图标颜色
- **主流程**：
  1. 搜索框圆角从 8px 增大到 `border-radius: 12px`
  2. 聚焦态外发光颜色与主色渐变对齐
  3. 搜索图标颜色从 `#9ca3af` 调整为稍深的中性色
  4. 清除按钮增加圆形背景 hover 态
- **异常流程**：无
- **验收标准**：
  - 搜索框宽度不变（240px 桌面 / 100% 移动端）
  - 聚焦动画与输入框一致
  - 清除按钮 hover 可见

### 3.9 优先级选择器 (PrioritySelector) 视觉升级

- **描述**：优化分段控制器的选中态样式，增加过渡动画
- **主流程**：
  1. 容器背景从 `#f9fafb` 调整为更柔和的浅灰
  2. 选中态背景增加轻微阴影
  3. 切换时增加背景色滑块动画（transform translateX）
  4. 优先级色块增加左侧小圆点指示器
- **异常流程**：无
- **验收标准**：
  - 选中态与三种优先级色系保持一致
  - 切换动画 200ms
  - 不影响 radiogroup/aria 语义

### 3.10 空状态 (EmptyState) 视觉升级

- **描述**：重新设计空状态页面，增加渐变背景、更精致的插图和更好的文字排版
- **主流程**：
  1. 空状态区域背景从纯白改为极淡渐变
  2. 插图区域从 SVG 线条画升级为带色彩的简化插画风格
  3. 标题字号增大（从 18px 到 20px），增加行间距
  4. 副标题颜色从灰色调整为品牌色系
  5. 整体增加一个淡入动画（首次加载时 opacity 0 -> 1）
- **异常流程**：无
- **验收标准**：
  - 空状态占满列表区域可用空间
  - 插图在 300ms 内淡入
  - 文字层级清晰（标题 > 副标题）
  - 移动端插图缩小但文字不变

### 3.11 骨架屏 (Skeleton) 视觉升级

- **描述**：优化骨架屏的形状和闪烁动画
- **主流程**：
  1. 骨架屏圆角与任务卡片保持一致（12px）
  2. 闪烁渐变从灰色调整为品牌色极淡渐变
  3. 每行骨架高度有微小差异模拟真实内容
- **异常流程**：无
- **验收标准**：
  - 骨架屏在 1s 内开始显示
  - 闪烁频率适中（1.5s 周期）
  - 降级优雅（动画性能差的设备仍可正常显示）

### 3.12 错误提示 (Error Toast) 视觉升级

- **描述**：优化错误提示的圆角、阴影和动画
- **主流程**：
  1. 圆角从 8px 增大到 12px
  2. 阴影加深并增加模糊半径
  3. 出现/消失增加 translateY 动画（从上方 -20px 滑入）
  4. 关闭按钮增加圆形 hover 背景
- **异常流程**：无
- **验收标准**：
  - toast 从顶部滑入，300ms 完成
  - 3s 后自动消失
  - 关闭按钮点击区域 >= 24x24px

### 3.13 底部输入区 (Input Section) 视觉升级

- **描述**：优化底部输入区的视觉分离效果
- **主流程**：
  1. 上边框从 1px solid 改为渐变分隔线或阴影分隔
  2. 背景从纯白改为极淡的毛玻璃效果
  3. 输入区与列表区的间距加大（从 16px 到 20px）
- **异常流程**：`backdrop-filter` 不支持的浏览器降级为纯色背景
- **验收标准**：
  - 输入区与列表区视觉分离清晰
  - sticky 效果正常
  - 毛玻璃降级方案可用

### 3.14 全局滚动条美化

- **描述**：优化自定义滚动条样式，使其更圆润柔和
- **主流程**：
  1. 滚动条宽度从 8px 缩小到 6px
  2. 滑块圆角增大到 `border-radius: 3px`
  3. 滑块颜色从 `#e5e7eb` 调整为更柔和的中性色
  4. hover 态滑块颜色使用主色系
- **异常流程**：Firefox 不支持 `::-webkit-scrollbar`，忽略即可
- **验收标准**：
  - 滚动条在 Chrome/Safari/Edge 中样式一致
  - 不影响滚动性能
  - Firefox 使用默认滚动条（可接受）

### 3.15 移动端响应式体验提升

- **描述**：针对移动端布局进行专项优化
- **主流程**：
  1. header 渐变在移动端保持，但高度适当压缩
  2. 任务卡片间距微调，增加横向内边距呼吸感
  3. 底部输入区增加上圆角（border-radius: 16px 16px 0 0）
  4. 操作按钮始终可见（已实现），增加触觉反馈视觉（按下时背景色变化）
- **异常流程**：无
- **验收标准**：
  - 移动端（< 768px）整体布局紧凑但不拥挤
  - 所有可交互元素最小触控区域 44x44px
  - 无水平滚动溢出
  - header 不超过两行高度

## 4. 非功能需求

### 4.1 性能

- 所有新增动画不得导致帧率低于 60fps（在中等性能设备上）
- `backdrop-filter` 毛玻璃效果不得导致滚动卡顿
- CSS 变量更新不得增加首屏渲染时间超过 100ms
- 所有动画必须使用 `transform` / `opacity` 等 GPU 加速属性，不使用 `width`/`height`/`top`/`left` 动画

### 4.2 可访问性

- 所有颜色变更后，文字与背景对比度仍满足 WCAG AA 标准（>= 4.5:1）
- 动画遵循 `prefers-reduced-motion` 媒体查询，该设置下禁用非必要动画
- focus-visible 样式在所有交互元素上保持可见
- 不引入纯装饰性动画导致注意力分散

### 4.3 兼容性

- Chrome、Firefox、Safari、Edge 最新两个主版本
- `backdrop-filter` 不支持的浏览器（如 Firefox < 103）提供降级方案
- 移动端 Safari 和 Chrome 测试通过
- 不引入新的 JS 依赖（纯 CSS 实现）

### 4.4 代码质量

- 所有颜色值通过 CSS 变量引用，`variables.css` 是唯一颜色定义入口
- 组件 `<style scoped>` 内不出现硬编码色值
- 新增动画使用 CSS `transition` / `animation` / `@keyframes`，不使用 JS 动画库
- 动画时长统一使用 CSS 变量（`--duration-*`），不内联写死

## 7. 假设与约束

### 7.1 假设

- [假设 1] v0.3.0 纯视觉升级不涉及组件结构（template/script）变更，仅修改 `<style>` 部分和 `variables.css`
- [假设 2] 主色渐变方向和色值由 UX/UI 设计师在 DES-301 中确定，前端不做设计决策
- [假设 3] 毛玻璃效果（`backdrop-filter: blur()`）在现代浏览器中性能可接受，不需要引入额外 polyfill
- [假设 4] 不引入新的 npm 依赖，所有视觉效果通过原生 CSS 实现
- [假设 5] 原规划 v0.3.0 的"虚拟滚动/性能优化"需求推迟至后续版本

### 7.2 约束

- [技术约束] 不改变组件 template 和 script 逻辑，仅修改样式
- [技术约束] 不引入 CSS-in-JS 或 CSS 框架（如 Tailwind），保持纯 CSS + CSS 变量方案
- [兼容约束] `backdrop-filter` 必须提供降级方案
- [动画约束] 所有动画使用 `transform`/`opacity`，不触发 layout/paint
- [时间约束] 预计开发周期 3-5 天

## 8. 验收标准总结

### 8.1 核心验收项

1. 全局配色变量更新完成，无硬编码色值残留
2. Header 具备渐变背景和毛玻璃效果（含降级方案）
3. 任务卡片具备左侧优先级竖条、增大圆角和柔和阴影
4. 复选框为圆形，选中时有弹性动画
5. 添加按钮使用渐变背景，点击有弹性反馈
6. 输入框和搜索框圆角增大，聚焦光效明显
7. 过滤器 Tab 为胶囊样式，选中态有背景色填充
8. 空状态有渐变背景和淡入动画
9. 移动端布局紧凑、触控区域合规
10. `prefers-reduced-motion` 下禁用非必要动画

### 8.2 不验收项

- 虚拟滚动和性能优化
- 深色模式
- 拖拽排序
- 任何功能逻辑变更

## 9. 风险与依赖

### 9.1 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 毛玻璃效果在低端移动设备上卡顿 | 移动端体验 | 降级为半透明纯色背景，性能检测后动态切换 |

| 过滤器滑块动画实现复杂度超预期 | 开发进度 | 降级为简单的背景色过渡（去掉 transform 滑块） |
