## 三、技术实现

### 3.1 项目结构

```
src/frontend/
├── src/
│   ├── components/      # 5 个 Vue 组件
│   ├── stores/          # 2 个 Pinia Store
│   ├── types/           # 2 个类型定义文件
│   ├── utils/           # 3 个工具函数模块
│   ├── assets/styles/   # 3 个样式文件（设计系统）
│   ├── App.vue          # 根组件
│   └── main.ts          # 应用入口
├── index.html           # HTML 入口
├── vite.config.ts       # Vite 配置
├── tsconfig.json        # TypeScript 配置
└── package.json         # 依赖管理
```

**文件统计**:
- Vue 组件: 5 个
- Store: 2 个
- 工具函数: 3 个模块
- 类型定义: 2 个文件
- 样式文件: 3 个文件
- 配置文件: 6 个

### 3.2 状态管理 (Pinia)

**任务 Store (task.ts)**:
- State: `tasks`, `loading`, `error`
- Getters: `completedTasks`, `uncompletedTasks`, `taskCount`, `uncompletedCount`
- Actions: `loadTasks`, `addTask`, `updateTask`, `deleteTask`, `toggleTaskStatus`, `clearAllTasks`

**过滤器 Store (filter.ts)**:
- State: `currentFilter`
- Getters: `filterLabel`
- Actions: `loadFilter`, `setFilter`

### 3.3 组件架构

```
App.vue (根组件)
├── FilterTabs.vue (过滤器)
└── TaskList.vue (任务列表容器)
    ├── TaskItem.vue (单个任务卡片) × N
    └── EmptyState.vue (空状态提示)

TaskInput.vue (任务输入框，固定在底部)
```

**组件通信**:
- Props: 父传子（任务数据、过滤器状态）
- Emits: 子传父（更新事件）
- Store: 全局状态共享（Pinia）

### 3.4 数据持久化

**LocalStorage Schema**:
```typescript
{
  tasks: Task[],
  version: string
}
```

**Storage Keys**:
- `todo_app_tasks`: 任务列表
- `todo_app_filter`: 过滤器状态
- `todo_app_version`: Schema 版本

**数据验证**:
- 加载时验证数据格式
- 保存时自动序列化
- 错误处理 + 降级方案

### 3.5 样式系统

**CSS Variables** (按设计规范实现):
- 色彩系统: 主色、辅助色、中性色、语义色、优先级色
- 字体系统: 字号 (12px ~ 24px)、字重 (400 ~ 700)
- 间距系统: XS (4px) ~ XXL (48px)
- 圆角系统: XS (2px) ~ Full (9999px)
- 阴影系统: Level 1 ~ Level 3
- 动画系统: 快速 (100ms) ~ 慢速 (300ms)

**响应式断点**:
- Mobile: < 768px
- Desktop: >= 768px

## 六、无障碍支持

### 6.1 键盘导航

- Tab: 在任务卡片、输入框、按钮间切换
- Enter: 提交表单 / 确认操作

### 6.2 ARIA 标签

- `role="checkbox"` + `aria-checked`: 复选框

### 6.3 屏幕阅读器

- 状态变化通过 `aria-live` 通知
